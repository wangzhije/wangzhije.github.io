Semver 语义化规范
  
## 语义版本控制标准 

`主版本.次版本.补丁版本`

- 主版本：当进行不兼容的 API 更改时，则升级主版本
- 次版本：当以向后兼容的方式添加功能时
- 补丁版本：当进行向后兼容的缺陷修复时
 
## 符号

- `^`：只会执行不更改最左边非零数字的更新
- `~`：更新到补丁版本
- `>`：接受高于指定版本的任何版本
- `>=`: 接受等于或高于指定版本的任何版本
- `<=`: 接受等于或低于指定版本的任何版本
- `<`: 接受低于指定版本的任何版本
- `=`: 接受确切的版本
- `-`: 接受一定范围的版本
- `||`: 组合集合
- `无符号`: 仅接受指定的特定版本
- `latest`: 使用可用的最新版本
